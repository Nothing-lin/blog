数组的概述

Array数组是多个相同类型的数据 – 按照一定顺序进行排列的集合，这个集合并且使用一个名字来对其进行命名，还有通过编号的方式来对这些数据进行统一的管理。

数组的重要元素

  • 数组名
  • 下标（或索引）
  • 元素
  • 数组的长度

数组本身是引用数据类型，而数组中的元素可以是任意的数据类型，包括基本数据类型和引用数据类型。

创建数组对象会在内存中开辟一整块连续的空间，而数组名引用的是这块连续空间的首地址。

数组的长度一旦确定下来的话就不能对其进行修改了。

我们可以直接通过下标或索引的方式来调用指定位置上的元素。

数组的分类：

  • 按照难度：一维数组、二维数组、三维数组·····
  • 按照类型：基本数据类型元素的数组，引用数据类型元素的数组（对象数组）

一维数组的使用（声明）：

一维数组的声明方式： type var[ ]; 或 type[ ] var;

int a[];
int[] a1;
double b[];
String[] c; //引用类型的变量数组

注意：

Java语言中声明数组时不能够指定其数组的长度（也就是数组元素中的数）

声明：int[5] a; ❌

要区分声明和定义，声明的过程中不可以定义数组的长度，但是在定义数组的过程中是可以这样进行定义的。

定义：int[5] a; ✔

一维数组的使用

初始化

  • 动态初始化
  • 静态初始化

动态初始化是指数组的声明过程中为数组元素分配空间和赋值操作是分开进行的。注意，数组的下标索引是从0开始的。

静态初始化是指在定义数组的同时就为数组元素分配空间和进行赋值操作。

数组元素的引用

对于数组元素是需要对数组进行定义并且使用运算符new为数组分配空间之后才可以对引用数组中的每个元素进行引用。

数组元素的引用方式：数组名【数组元素下标】array[ 3 ]

数组元素的下标是可以整型或者整理表达式进行表达的。a[3]、b[i]、c[6*i]。

数组元素下标是从0开始的，数组的长度为n，那么数组的合法下标的取值范围是[0,n-1]。例如：int[3];的可引用数组元素为a[0]、a[1]、a[2]。

每一个数组都有一个属性length来指明它的长度，例如：a.length指明数组a[ ]的长度，也就是素组中元素的个数。

数组元素的默认初始化值

数组是引用类型，数组的元素相当于类的成员变量，因此数组一经分配空间，其中的每个元素也被按照成员变量同样的方式被隐式初始化。

public class Test{
    public static void main(String arg[]){
        int a[]=new int[5];
        System.out.println(a[3]); // a[3]的默认值为0
    }
    
}
  • 对于基本数据类型而言，默认初始化值各不相同
  • 对于引用数据类型而言，默认初始化值为null

创建基本数据类型数组

Java中使用关键字new来创建数组，如下：

public class Test{
    public static void main(String[] args){
        int[] s;
        s = new int[10];
        for(int i = 0; i <10; i++){
            s[i] = 2*i+1;
            System.out.println(s[i]);
        }
    }
}

多维数组的使用

Java语言提供了支持多维数组的语法，多维数组可以理解成为一个表格，一维数组可以理解为这个表格中的一个记录。

对于二维数组的理解，我们可以看成是一维数组中有放了另一个一维数组。从数组底层的运行机制来看，其实没有多维数组。

二维数组的使用

动态初始化：int[ ][ ] arr = new int[3][2];

上面的式子中定义了一个名称为arr的二维数组，这个二维数组中有3个一维数组，每一个一维数组中有2个元素。这里的一维数组名称分别为arr[0]，arr[1]，arr[2]，如果要给第一个一维数组为1的脚标赋值为78的写法是：arr[0][1]。

二维数组的动态定义

注意：

在二维数组中，每一个一维数组都是一个默认初始化值为null值的，可以分别对这三个一维数组分别进行初始化，如下：

arr[0] = new int[3]; //定义二维数组第一个一维数组可以容纳3个元素
arr[1] = new int[1]; //定义二维数组第二个一维数组可以容纳1个元素
arr[2] = new int[2]; //定义二维数组第三个一维数组可以容纳2个元素

错误示范：

int [][] arr = new int[][3] //这种二维数组初始化是错误的

像这种没有初始化二维数组的一维数组的个数，默认情况下是null也就是默认情况下没有初始化二维数组中的一维数组的话，上面定义某个一维数组的容纳个数为3，那么试问这个3是给第几个一维数组进行定义的？

这就导致了歧义，所以我们需要注意上面这种错误的二维数组定义。

二维数组的静态定义

int[][] arr = new int[][]{{3,8,2},{2,7},{9,0,1,6}}

上面的静态数组定义是定义了一个名称为arr的二维数组，这个二维数组中有三个一维数组，对于每一个一维数组中具体元素也都已进行了初始化。

  • 第一个一维数组arr[0] = {3,8,2};
  • 第二个一维数组arr[1] = {2,7};
  • 第三个一维数组arr[2] = {9,0,1,6};
  • 第三个一维数组的长度表示方式：arr[2].length;

二维数组的特殊写法

int[]x,y[];

在这里，x是一维数组，y是二维数组。

排序算法

排序是指假设含有n个记录的序列为{R1，R2，···，Rn}，对应关键字序列为{K1，K2，···，Kn}。将这些记录重新排列为{Ri1，Ri2，···，Rin}，使得相应的关键字（下标索引）满足Ki1 <= Ki2 <= … <= Kin。这样的操作就叫排序。

上面的定义太官方了，也就是说有一个数组{8，9，7，4，5}，现在要将它们从小到大进行排序，素组元素对应的位置分别为1，2，3，4，5。现在我们需要将{8，9，7，4，5}变为{4，5，7，8，9}这个数组的过程就叫做排序。

通常来说，排序的目的是为了能够快速查找

衡量排序算法的优劣

  • 时间复杂度，分析关键字的比较次数和总移动次数

关键字是什么，就是素组中元素的值

  • 空间复杂度，分析排序算法中需要多少内存
  • 稳定性，如果两个记录A和B的关键字的值相等，但是排序后A、B的先后次序保持不变，那么就称这种排序算法是稳定的。

排序算法的分类

  • 内部排序

内部排序是指在排序过程中不需要借助外部存储器（如磁盘等），所有的排序操作都是在内存中进行完成的。

  • 外部排序

外部排序参与排序的数据非常多，数据量非常大，计算机无法把整个排序过程放在内存中完成，必须借助于外部存储器（如磁盘）。外部排序最常见的是多路归并排序，可以认为外部排序是由多次内部排序组成的。

算法的5大特征

  • 输入input

有0个或多个输入数据，这些输入必须有清楚的描述和定义

  • 输出Output

至少有1个或多个输出结果，不可以没有输出结果

  • 有穷性（Finiteness）

算法在有限的步骤之后会自动结束而不会无限循环，并且每一个步骤可以在可接受的实践内完成

  • 确定性（definiteness）

算法中的每一步都有确定的含义，不会出现二义性

  • 可行性（effectiveness）

算法的每一步都是清楚可行的，能让用户用纸笔计算而求出答案
