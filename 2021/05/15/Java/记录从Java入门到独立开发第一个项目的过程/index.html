开始

从大一开始进入编程领域到现在，有效成长时长大概有一年了。现在是大二接近尾声，即将迎来大三的时光。总结大一的折腾时光无非是在对编程道路的探索，接触了博客搭建，服务器的概念理解，编程应用领域以及未来可发展道路。前面一年所有的探索都是为后面的成长做铺垫，主要是那一年极大地提升了自己对编程的认识和热爱。哦，原来我可以编写程序来解决这些问题。哦，原来我可以编写程序帮别人完成这些问题，等等…

我现在最大的愿望就是能够希望自己能够达到这种境界，就是我能够开发出一些编程工具来解决我目前的需求，比如英语学习插件，或者个人的笔记平台，不止这些，看日后更多的需求。也就是我能够想得到的我都能做得到，就是这样子。

我自己总结了一下自己的学习方式，我很难接受被动学习。而对于我来说，被动学习是指这种情况：

一开始学习一门编程语言就直接拿起厚厚的书来一页一页地看，对于书中不懂得概念一定要弄懂，假设把书看完之后却不知道自己学过什么，能够做什么。

就像我大一刚接触C语言得时候，我并不知道这能干什么。我拿着学校得书本一页一页地看，但我知道自己看不下去，于是对书中地内容一页一页地抄那些概念。最后却浪费了非常多时间并且中途放弃。那个时候是我最痛苦地时候，我被这种被动学习折磨得痛苦得快哭了，并且那时候看社区话题都在谈论程序员中年危机，我记得我当时迷茫到极致了。

到现在我对于自己在什么情况下能够进行高效学习有了自己的判断，我的学习方法就是摸石头过河。

很多人总是建议我们这些新手先去打好基础在去接触项目开发。根据我自己的实践证明，这种方法只会让我放弃学习。因为我看着这些编程书籍，看完一大堆概念之后又把概念给忘光，后面接触现目开发的时候又要重新去学习一遍，这样很难激发我的学习热情。反而我学习的目的是为了完成一个项目的全部功能来说，我就觉得自己学有所用，这样我的学习热情提升至了顶点。我想说的是，书籍同样非常重要，但是我们先要知道这些知识能够运用到那些场景中去。比如说，我刚接触一个登录功能的实现，那么我该如何实现这个功能呢？我在GitHub找了份能够学习的代码进行分析，找到了相关的代码结构和代码函数。这时候我们没有基础肯定很难理解，所以我们就分析代码所涉及到的知识，去网上获取答案或者在书籍中获取答案，然后记录下来，就这样一个问题一个问题的积累，反而在提高我们编程热情的同时也能够帮我们打好基础。我们学习一门知识的前提需要我们对这门知识能够有热情，而有热情的前提又需要我们足够了解这门知识的应用场景在什么地方。

整一套下来只是针对我个人的学习方法论，但我个人觉得我自己的学习方法还不是很完善，但确定的就是这种摸石头过河的学习方式。
